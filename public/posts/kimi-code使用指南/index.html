三月底在GLM、MiniMax、Kimi几个选项中试了一周 Kimi Code。说实话我一开始没抱太大期望——Claude、Cursor、Codex这类产品能力确实强，但是因为网络和支付限制订阅麻烦，而且价格不便宜，用的时候缩手缩脚。国内这三家负面评价不少，宣传语和最终体验之间通常隔着一层。但Kimi Code有两个蛮好的：一个是原生Agent，命令行、vscode插件、Web UI界面、自定义skill都有；另一个是从安装订阅到开始使用，整个流程非常丝滑，一两分钟就能开始用。能力谈不上惊艳，但在我常用的场景里够用且稳定，这就是我愿意掏钱的标准。这个月初直接买了包年。

但公平地说，买了之后这半个月，我大概只用了它 60% 的表面功能，更别说摸清它和终端、Git 工作流之间的最优配合方式。所以这篇更像一份我自己正在建立的实操手册：从官方文档和零散实验中筛出我真正反复用的部分（安装、交互模式、快捷键、命令速查），加上我自己踩过的小坑。如果你也在评估值不值得跳槽过来，或者买了但还没深用，这篇可能会省你点时间。

简介#

Kimi Code CLI 是一个运行在终端中的智能AI助手（AI Agent），它能够阅读和编辑代码、执行Shell命令、搜索网页，并自主规划和完成任务。

适用场景：

  • 代码开发：编写新功能、修复Bug、代码重构。
  • 项目理解：探索陌生的代码库，解答关于项目架构和逻辑的问题。
  • 任务自动化：批量处理文件、运行构建脚本、执行测试等重复性任务。

三种使用方式：

方式 命令 说明
交互式终端 kimi 在命令行界面中以对话方式进行交互
浏览器界面 kimi web 在浏览器中打开图形化的Web UI进行操作
服务集成 kimi acp 作为后台服务嵌入到IDE等客户端中使用

安装与升级#

系统要求：Python 3.12 至 3.14（推荐使用 3.13）。

安装命令：

  • Linux / macOS:
1curl -LsSf [https://code.kimi.com/install.sh](https://code.kimi.com/install.sh)| bash
  • Windows (PowerShell):
1Invoke-RestMethod [https://code.kimi.com/install.ps1](https://code.kimi.com/install.ps1)| Invoke-Expression

验证与维护：

1# 验证安装
2kimi --version
3# 升级 CLI
4uv tool upgrade kimi-cli --no-cache
5# 卸载 CLI
6uv tool uninstall kimi-cli

注意：在macOS上首次运行时可能较慢。你可以在「系统设置 → 隐私与安全性 → 开发者工具」中添加你的终端应用程序（如Terminal或iTerm2），以获得最佳性能。

快速开始#

第一步：进入项目目录#

1cd your-project
2kimi

第二步：首次配置#

  1. 启动后，输入命令 /login。
  2. 选择你使用的平台：
    • Kimi Code：将自动在浏览器中打开OAuth授权页面，按引导登录即可。
    • 其他平台：需要手动输入你的API密钥，并从列表中选择要使用的模型。

第三步：开始对话#

配置完成后，你就可以直接用自然语言描述你的任务了，例如“帮我看看这个项目的入口文件是哪个？”。

提示：如果项目目录下没有 AGENTS.md 文件，可以运行 /init 命令，CLI 会为你自动生成项目说明，这有助于AI更好地理解你的代码库。

核心交互模式#

1. 两种基础模式#

默认启动后，你会处于 Agent 模式，你的输入会被发送给AI处理。你可以随时切换到 Shell 模式，在此模式下你的输入会直接作为本地Shell命令执行。

模式 说明 切换方式
Agent 模式 (默认) 与AI对话，由AI处理复杂任务。 Ctrl-X
Shell 模式 直接执行本地Shell命令。 Ctrl-X

注意：Shell模式下，每次命令都是独立执行的。例如，在Shell模式执行 cd src 后，后续的命令依然在原始目录下运行，不会保留状态。对于需要保持状态的Shell操作，请在Agent模式中让AI来执行。

2. Plan 模式 (规划模式)#

在Plan模式下，AI只能使用只读工具（如查看文件、读取目录）来探索你的代码库。它会制定一个详细的实施方案并写入到 plan 文件中，等待你的审批。只有在你批准后，AI才会开始执行写文件、运行命令等实际变更操作。

进入方式：

  • 启动时进入：kimi --plan
  • 交互中进入：按 Shift-Tab 键，或输入命令 /plan、/plan on
  • AI请求进入：当AI认为任务复杂时，可能会请求进入Plan模式，你可以选择同意或拒绝。

审批选项： 当你看到一个计划时，会看到以下选项：

选项 说明
批准 同意AI按选定方案执行。
拒绝 拒绝此计划，但仍保持在Plan模式，你可以提供反馈让AI重新规划。
拒绝并退出 拒绝此计划，并退出Plan模式，返回到常规Agent模式。
修订 输入你的修改意见，AI会基于你的反馈重新修订计划。

提示：在Plan模式下，终端的提示符会变成 📋，并且在状态栏会显示蓝色的 plan 标识。

3. Thinking 模式 (思考模式)#

此模式强制AI在回答前先进行深入思考，将思考过程（Reasoning）也显示给你。这有助于处理非常复杂、需要多步推理的问题。

开启方式：

  • 启动时开启：kimi --thinking
  • 切换模型时开启：使用 /model 命令切换模型时，可以指定开启Thinking模式。
  • 特定模型：部分模型（如 kimi-k2-thinking-turbo）默认就启用了思考功能。

键盘快捷键速查#

快捷键 功能
Ctrl-X 在 Agent 模式和 Shell 模式之间切换。
Shift-Tab 在普通模式和 Plan 模式之间切换。
Ctrl-O 在外部编辑器中打开一个临时文件来编辑多行输入。
Ctrl-J / Alt-Enter 在当前输入中插入一个换行符，用于输入多行文本。
Ctrl-S Steer：在AI流式输出过程中，立即打断并注入新消息。
Ctrl-V 从剪贴板粘贴（支持文本、图片、视频）。
Ctrl-E 在审批请求面板中，展开/折叠被隐藏的详细内容。
1 / 2 / 3 在审批面板中，快速选择对应编号的选项。
4 在审批面板中，快速选择“附带反馈拒绝”。
Ctrl-C 中断AI的当前输出或正在运行的命令。
Ctrl-D 退出CLI（仅在输入框为空时生效）。

斜杠命令速查#

输入 / 后按 Tab 键会自动补全命令，支持模糊匹配。

账号与配置#

命令 功能
/login 登录或配置新的API平台/密钥。
/logout 登出当前已登录的平台。
/model 查看可用模型列表，或切换模型及Thinking模式。
/editor 设置用于编辑的外部编辑器（如 vim, code）。
/theme 在深色和浅色终端主题之间切换。
/reload 重新加载配置文件。

会话管理#

命令 功能
/new 创建一个全新的会话。
/sessions 或 /resume 列出所有可恢复的会话，并选择一个切换过去。
/title <text> 为当前会话设置一个标题，便于识别。
/undo 回退到对话历史中的某一轮，并基于此状态“重试”一个新会话。
/fork 复制当前会话的所有上下文到一个全新的会话中。
/export [path] 将会话历史导出为Markdown文件。
/import <file或session_id> 导入一个Markdown文件或指定会话ID的历史。
/clear 或 /reset 清空当前会话的对话上下文。
/compact [提示] 手动压缩对话上下文，可附加提示（如“保留数据库讨论”）来保留关键信息。

工作模式#

命令 功能
/plan 切换 Plan 模式的开关状态。
/yolo 切换 YOLO 模式（自动批准所有操作）的开关状态。
/task 打开后台任务浏览器，查看和管理正在运行或已完成的任务。
/btw <问题> 侧问：提出一个新问题，但不会中断当前的对话流。

Skills 与工作区#

命令 功能
/skill:<name> 加载并使用指定的 Skill 插件。
/flow:<name> 加载并执行指定的 Flow Skill。
/add-dir [path] 为AI添加一个额外的工作区目录，使其能够访问该目录下的文件。

其他#

命令 功能
/help 显示所有可用命令和已加载Skills的帮助信息。
/web 切换到Web UI界面。
/vis 打开Agent执行追踪的可视化工具。
/init 在项目中生成一个 AGENTS.md 文件。
/debug 显示调试信息。
/usage 显示当前API的使用统计。

会话管理#

续接历史会话#

  • 继续最近会话：kimi --continue
  • 交互式选择会话：kimi --session
  • 恢复指定会话：kimi -r <session-id>

在CLI运行过程中，你也可以随时使用 /sessions 命令切换到其他历史会话。使用 /title 命令为重要会话命名，能帮助你更好地查找和管理它们。

持久化状态#

当你恢复一个会话时，以下状态会被自动还原：

  • YOLO模式开关状态和审批决策记录
  • Plan模式的开关状态
  • 创建的子Agent实例
  • 通过 /add-dir 命令添加的额外工作区目录

上下文管理#

  • 查看状态：底部状态栏会显示 context: 使用率 (token数/上限)。
  • 自动压缩：当对话上下文过长，即将超过模型的token限制时，CLI会自动进行压缩。
  • 手动压缩：你可以通过 /compact 命令主动压缩上下文。可以附加提示，例如 /compact 保留数据库讨论，以告诉AI在压缩时重点保留哪些信息。

审批、YOLO 与后台任务#

审批确认#

当AI计划修改文件或执行Shell命令时，会向你请求确认。

选项 说明
允许 批准执行当前这一个操作。
本会话允许 在当前整个会话期间，自动批准所有同类操作。
拒绝 / 附带反馈拒绝 拒绝执行。选择后者可以向AI提供拒绝的理由。

YOLO 模式#

在YOLO模式下，AI会自动跳过所有需要用户确认的步骤。请谨慎使用，建议在临时的、可恢复的隔离环境中尝试。

开启方式：

  • 启动时开启：kimi --yolo
  • 运行中开启：输入命令 /yolo

后台任务#

AI可以将一些耗时的命令（如 npm run build, python -m pytest）放入后台运行，这样你无需等待，可以继续和AI进行其他对话。

  • 管理任务：输入 /task 打开任务浏览器，可以查看所有后台任务的状态、标准输出和错误信息。
  • 自动处理：当后台任务完成后，如果AI当前处于空闲状态，会自动触发新一轮对话来处理任务结果。

常见使用案例#

场景 示例指令
实现新功能 “给 user/list 页面添加分页功能，每页显示 20 条记录。”
修复Bug “运行 npm test 时出现 TypeError: Cannot read property 'map' of undefined，帮我定位并修复这个问题。”
理解项目 “这个项目的整体架构是怎样的？主入口文件在哪里？”
自动化任务 “把 src 目录下所有 .js 文件里的 var 关键字改成 const 或 let。”
添加文档 “给 utils 模块下所有没有写docstring的公开函数添加文档注释。”
技术调研 “帮我调研Python的异步Web框架，比较FastAPI和Starlette在性能和易用性上的优缺点。”
数据分析 “分析 logs/access.log 文件，统计每个API接口在过去一天的调用次数和平均响应时间。”
